指定したディレクトリ（省略時は現在のディレクトリ）を起点に `claude --background` で独立した Claude Code セッションを起動し、渡された作業を委譲します。別リポジトリを対象に起動することもできます。起動したセッションは agent view（`claude agents`）で管理でき、サブエージェントとは異なり独立したセッションとして扱われます。

このスキルは**ディスパッチャに徹する**。依頼内容を解釈・展開したり、関連ファイルを探しに行ったりはせず、ユーザーが渡した依頼テキストをそのまま background セッションに横流しする。実際の解釈と作業は起動先のセッションに任せる。

## 手順

1. `/delegate [対象ディレクトリ] <依頼>` の引数を受け取る。
   - 先頭の引数が対象ディレクトリのパス（`/`・`~`・`./` で始まる、または既存ディレクトリを指す）であれば、それを起点ディレクトリとして扱い、残りを**依頼テキスト**とする。
   - パス指定がなければ現在のディレクトリを起点とする。
   - 依頼テキストに評価・1on1（人事評価、フィードバック、目標設定、1on1 の準備・記録など）に関する語が含まれる場合は、パス指定の有無に関わらず**必ず** `$HOME/.obsidian/igsr5/10_performance_review` を起点ディレクトリとする。
2. 対象ディレクトリを指定された場合は、それが実在するディレクトリかを確認する（存在しなければユーザーに知らせて止まる）。
3. 依頼テキストは**加工せずそのまま**渡す。背景の補足・context の埋め込み・「自己完結したプロンプトへの整形」は行わない。ユーザーが書いた文字列がそのまま background セッションのプロンプトになる。
4. 対象ディレクトリで `claude --background` を起動する。
   - 現在のディレクトリが対象なら `claude --background "<依頼テキスト>"`。
   - 別ディレクトリが対象なら `(cd <対象ディレクトリ> && claude --background "<依頼テキスト>")`。
   - 依頼テキストはシェルに渡すため、含まれる二重引用符やバッククォート等の**エスケープのみ**行う（内容は変えない）。
5. 起点ディレクトリ・出力されたセッション ID・管理コマンドをユーザーに報告する。

## 注意事項

- 依頼テキストは横流しするだけで context は引き継がれない。「さっきの件」「上記の」といった指示語が含まれていても**こちらで展開・補完はしない**。曖昧さの解消は起動先セッションに委ねる（起動先が対象ディレクトリを調べて判断する）。
- push・PR 作成・デプロイなど外部影響や破壊的操作を委譲する場合でも、依頼テキストは加工しない。ガードレールが必要だと判断したら、プロンプトを書き換えるのではなく事前にユーザーへ確認する。
