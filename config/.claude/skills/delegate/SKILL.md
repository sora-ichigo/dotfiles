指定したディレクトリ（省略時は現在のディレクトリ）を起点に `claude --background` で独立した Claude Code セッションを起動し、渡された作業を委譲します。別リポジトリを対象に起動することもできます。起動したセッションは agent view（`claude agents`）で管理でき、サブエージェントとは異なり独立したセッションとして扱われます。

## 手順

1. `/delegate [対象ディレクトリ] <ざっくりした依頼>` の引数を受け取る。
   - 先頭の引数が対象ディレクトリのパス（`/`・`~`・`./` で始まる、または既存ディレクトリを指す）であれば、それを起点ディレクトリとして扱い、残りを依頼内容とする。
   - パス指定がなければ現在のディレクトリを起点とする。
   - 依頼内容が空なら、何を委譲したいかユーザーに尋ねる。
2. 依頼内容に評価・1on1（人事評価、フィードバック、目標設定、1on1 の準備・記録など）に関する内容が含まれる場合は、パス指定の有無に関わらず**必ず** `$HOME/.obsidian/igsr5/10_performance_review` を起点ディレクトリとする。
3. 対象ディレクトリを指定された場合は、それが実在するディレクトリかを確認する（存在しなければユーザーに知らせて止まる）。
4. 依頼内容を、独立セッションが単体で完遂できる**自己完結したプロンプト**に整形する。
   - background セッションは今の会話の context を一切引き継がないため、必要な背景・対象ファイル・完了条件をすべてプロンプトに書き込む。
   - 目的が曖昧な場合は先にユーザーに確認する。
5. 対象ディレクトリで `claude --background` を起動する。
   - 現在のディレクトリが対象なら `claude --background "<整形したプロンプト>"`。
   - 別ディレクトリが対象なら `(cd <対象ディレクトリ> && claude --background "<整形したプロンプト>")`。
   - プロンプトはシェルに渡すため、含まれる二重引用符やバッククォート等を適切にエスケープする。
6. 起点ディレクトリ・出力されたセッション ID・管理コマンドをユーザーに報告する。

## 管理コマンド

- `claude agents` — セッション一覧（agent view）
- `claude logs <id>` — 直近の出力を確認
- `claude attach <id>` — このターミナルで開く
- `claude stop <id>` — 停止

## 注意事項

- context を引き継がないので、「さっきの件」「上記の」といった指示語はプロンプトに残さず、具体的に展開する。
- push・PR 作成・デプロイなど外部影響や破壊的操作を委譲する場合は、ガードレール（main/master へ直接コミットしない、force push/merge しない、ブランチや worktree で作業する等）をプロンプトに明記するか、事前にユーザーへ確認する。
- 委譲後は「起動しただけ」であり完了ではない。結果は `claude logs <id>` や agent view で確認する。
