作業が完了したワークツリーを削除し、セッションを元のディレクトリに戻して片付けます。

## 前提

- このセッションが `EnterWorktree` で作成したワークツリーで作業していること
- 作業内容（コミット）はすでにPRとしてマージ済み、または破棄してよい状態であること

## 手順

1. ワークツリーの状態を確認する
   - 未マージのコミットや未コミットの変更がないか `git status`, `git log` で確認する
2. ワークツリーのコミットがリモートにマージ済みか確認する
   - `gh pr view` でPRがMERGED状態か確認する
3. `ExitWorktree` でワークツリーを削除する
   - マージ済みで残コミットが破棄対象なら `action: "remove"`, `discard_changes: true`
   - まだ残したい変更があれば `action: "keep"` を使う
4. 元のディレクトリに戻ったことを確認する

## 重要な注意事項

- **マージ前の作業を誤って破棄しない**：`discard_changes: true` を使う前に、コミットがリモートに反映済みか必ず確認する
- マージが未完了の場合は、先にマージを済ませてから片付ける
- 残しておきたい変更がある場合は `action: "keep"` を選び、削除しない
